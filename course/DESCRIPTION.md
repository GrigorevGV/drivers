# Описание проекта курсовой работы

## Тема
Разработать драйвер для передачи CAN пакетов через Ethernet

## Цель работы
Создать модуль ядра Linux, который обеспечивает передачу CAN-фреймов через сеть Ethernet с использованием протокола UDP/IP.

## Задачи
1. Изучить архитектуру SocketCAN в Linux
2. Изучить механизмы работы с сетью в пространстве ядра
3. Разработать модуль ядра, связывающий CAN интерфейс и Ethernet
4. Реализовать инкапсуляцию CAN-фреймов в UDP пакеты
5. Протестировать работу драйвера

## Используемые технологии
- Ядро Linux
- SocketCAN API
- UDP/IP протокол
- Модули ядра Linux (kernel modules)
- Kthread для асинхронной обработки

## Структура проекта
- `can_ethernet.c` - основной код модуля ядра
- `Makefile` - файл для сборки модуля
- `README.md` - подробная документация
- `commands.txt` - команды для сборки и запуска
- `DESCRIPTION.md` - описание проекта (этот файл)

## Принцип работы
1. Модуль создает два сокета: CAN-сокет для приема фреймов и UDP-сокет для отправки через сеть
2. Запускается поток ядра (kthread), который непрерывно читает CAN-фреймы
3. Каждый полученный CAN-фрейм инкапсулируется в UDP-пакет и отправляется на указанный IP-адрес
4. Структура UDP-пакета включает CAN ID, DLC, флаги и данные

## Особенности реализации
- Использование стандартных API ядра Linux
- Обработка ошибок и логирование
- Настраиваемые параметры через параметры модуля
- Поддержка стандартных и расширенных CAN-фреймов

## Тестирование
Драйвер тестируется с использованием виртуального CAN-интерфейса (vcan) и реального CAN-адаптера. Для приема UDP-пакетов используется утилита netcat.

## Ограничения
- Односторонняя передача (CAN -> Ethernet)
- Использование UDP (без гарантии доставки)
- Требуются права root для загрузки модуля

## Дальнейшее развитие
- Реализация двусторонней связи (Ethernet -> CAN)
- Добавление TCP-протокола для надежной доставки
- Поддержка нескольких CAN-интерфейсов одновременно
- Улучшенная обработка ошибок и восстановление соединения
